# Security Policy

## Supported Versions

This is an early-stage project. Security fixes are provided on a best-effort basis for the latest main branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

1. Send a private report to the maintainers.
2. Include reproduction steps, environment details, and impact assessment.
3. If possible, include a minimal proof-of-concept.

We will acknowledge receipt as soon as possible and provide updates on triage and remediation.

## Scope Notes

This repository is a research-style, minimal CV demo. It should not be used directly in safety-critical production systems without additional hardening, validation, and threat modeling.
